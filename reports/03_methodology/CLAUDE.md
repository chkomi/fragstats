# 프로젝트 폴더 구조 가이드

이 문서는 본 연구 프로젝트의 일관성 있는 폴더 구조를 정의하고, 각 폴더의 역할과 저장되는 파일의 종류를 설명합니다. 모든 연구 참여자는 이 구조를 따름으로써 파일 관리의 효율성을 높이고 협업을 원활하게 할 수 있습니다.

---

## 최상위 폴더 구조

```
FRAGSTATS/
├── src/                # 모든 소스 코드
├── data/               # 원본 데이터
├── reports/            # 보고서 및 문서
├── results/            # 분석 결과물
├── paper/              # 참고 논문
├── tmp/                # 임시 파일
└── .gitignore          # Git 무시 파일 목록
```

---

## 하위 폴더 상세 설명

### 1. `src/` - 소스 코드

모든 Python 스크립트 및 소스 코드를 저장합니다. 역할에 따라 다음과 같이 분류합니다.

*   `src/01_data_preparation/`: 데이터 전처리, 형식 변환 등 분석 준비 단계에 사용되는 스크립트.
*   `src/02_analysis_models/`: 엔트로피 가중치 모델 등 **핵심 분석 모델**을 구현한 스크립트.
*   `src/03_verification/`: 계산 결과 검증, 보조 계산, 시각화 등 유틸리티 성격의 스크립트.

### 2. `data/` - 원본 데이터

분석에 사용되는 원본 데이터를 저장합니다.

*   `data/01_raw_fragstats/`: FRAGSTATS 프로그램에서 직접 산출된 `.txt` 형식의 원본 경관지표 데이터.

### 3. `reports/` - 보고서 및 문서

분석 과정 및 결과에 대한 모든 종류의 문서를 저장합니다.

*   `reports/01_manuscripts/`: 학회 제출 또는 최종 결과물로 작성되는 논문 원고 (`.hwpx`, `.docx` 등).
*   `reports/02_analysis_reports/`: 분석 결과를 정리하고 해석한 보고서 (`비교_분석_보고서.md` 등).
*   `reports/03_methodology/`: 연구 방법론, 지표 선정 근거, 계산 과정 등을 상세히 기술한 문서.

### 4. `results/` - 분석 결과물

스크립트 실행을 통해 생성된 모든 결과 파일을 저장합니다.

*   `results/01_initial_outputs/`: 초기 분석 또는 이전 버전의 모델로 생성된 결과 파일.
*   `results/02_toyang3_reanalysis/`: `toyang3` 데이터를 사용하여 재분석한 최신 결과 폴더. (향후 다른 분석 수행 시 `03_...` 와 같이 폴더 추가)

### 5. `paper/` - 참고 논문

연구에 참고한 모든 외부 논문, 기사, 기술 문서 등을 저장합니다.

### 6. `tmp/` - 임시 파일

HWPX 파일의 압축을 해제한 내용이나, 분석 과정에서 일시적으로 생성되었다가 삭제되어도 무방한 파일들을 저장합니다. 이 폴더의 내용은 Git 추적 대상이 아닙니다.

---

**규칙:**
- 모든 새로운 스크립트, 데이터, 보고서, 결과물은 반드시 위 구조에 맞게 해당 폴더에 저장합니다.
- 스크립트 내의 파일 경로는 절대 경로를 사용하지 않고, 스크립트의 위치를 기준으로 상대 경로를 계산하여 사용합니다.
- 폴더 구조에 대한 변경이 필요할 경우, 팀원과 논의 후 이 `CLAUDE.md` 문서를 먼저 업데이트합니다.
